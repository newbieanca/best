KONEKSI FIREBASE CLOUD

Sinkronisasi Real-time dengan Google Firebase Firestore

Cara Kerja Firebase Firestore:

Data Best Employee tersimpan di koleksi best_employees pada database Firestore milik proyek Firebase Anda. Semua terminal/TV yang menggunakan konfigurasi yang sama akan terhubung ke database yang sama secara real-time.

Konfigurasi disimpan lokal di browser ini saja. Lihat tab Tutorial untuk cara mendapatkan nilai-nilai di atas.

Tutorial ini akan memandu Anda membuat proyek Firebase baru dan menghubungkannya ke aplikasi ini secara gratis. Estimasi waktu: ±5 menit.

1

Buka Firebase Console

Pergi ke console.firebase.google.com dan login dengan akun Google Anda.

2

Buat Proyek Baru

→ Klik tombol "Add project" atau "Buat project"

→ Masukkan nama proyek, contoh: sharp-best-employee

→ Google Analytics: bisa dinonaktifkan (tidak wajib)

→ Klik "Create project" → tunggu hingga selesai → klik "Continue"

3

Daftarkan Web App

→ Di halaman Project Overview, klik ikon </> (Web)

→ App nickname: sharp-hof-display

→ Jangan centang Firebase Hosting

→ Klik "Register app"

→ Anda akan melihat blok kode firebaseConfig seperti ini:

// Salin nilai-nilai ini ke tab Konfigurasi const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX", authDomain: "sharp-best-employee.firebaseapp.com", projectId: "sharp-best-employee", storageBucket: "sharp-best-employee.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };

Copy nilai masing-masing field ke tab Konfigurasi di atas.

4

Aktifkan Cloud Firestore

→ Di panel kiri Firebase Console, klik Build → Firestore Database

→ Klik "Create database"

→ Pilih "Start in production mode" → klik Next

→ Pilih region terdekat (contoh: asia-southeast2 untuk Jakarta)

→ Klik "Enable" → tunggu proses selesai

5

Atur Firestore Security Rules (Izin Baca/Tulis)

→ Di halaman Firestore, klik tab "Rules"

→ Hapus aturan yang ada, ganti dengan kode berikut:

// Firestore Rules - izin baca & tulis terbuka // (cocok untuk jaringan internal pabrik) rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /best_employees/{docId} { allow read, write: if true; } } }

Catatan: Rules ini membuka akses untuk semua pengguna. Cocok untuk display internal pabrik. Untuk produksi publik, tambahkan autentikasi.

→ Klik "Publish"

6

Hubungkan ke Aplikasi Ini

→ Kembali ke tab Konfigurasi di atas

→ Isi semua field dengan nilai dari firebaseConfig yang tadi di-copy

→ Klik Hubungkan Firebase

→ Jika berhasil, indikator di header akan berubah menjadi CLOUD ACTIVE

Selesai! Semua data Best Employee kini tersimpan di cloud dan dapat diakses dari browser/PC manapun yang menggunakan konfigurasi yang sama.

7

Opsional: Lihat Data di Firebase Console

→ Buka Firestore Database → Data

→ Koleksi best_employees akan muncul setelah data pertama disimpan

→ Setiap dokumen mewakili satu catatan Best Employee

→ Anda bisa mengedit atau menghapus data langsung dari sini
